# Hearth Community Hub Design Book

## 1. Template Identity
Hearth Community Hub is a warm community platform homepage that helps members participate without feeling overwhelmed.

## 2. Brief Match
- Type: community platform.
- Layout: welcome hero, event calendar, discussion highlights, member spotlights, learning paths, and contribution prompts.
- UX: paths guide users to join a conversation, attend an event, and meet members.
- Responsive: mobile prioritizes today's events and active discussions.

## 3. Audience
Members, moderators, mentors, hosts, and community teams managing a living digital community.

## 4. Information Architecture
The page moves from welcome and participation paths into today's events, discussion highlights, member spotlights, learning paths, and prompts.

## 5. Layout System
Desktop uses a split welcome hero and balanced content grids. Mobile stacks the urgent participation paths first.

## 6. Responsive System
Below 760px the mobile priority chips appear, today's events come forward, and all grids become single-column.

## 7. Typography
Nunito Sans is used for approachable UI and body copy. Fraunces is used for warm editorial headings and key community labels.

## 8. Color System
- Cream: #FFF8EA
- Forest: #1F4D3A
- Coral: #FF6B6B
- Marigold: #FBBF24
- Sky: #BAE6FD
- Ink: #1F2937

## 9. Visual Language
The interface uses warm panels, handoff-like participation paths, soft connection lines, and member initials as friendly visual assets.

## 10. Components
Core components include sticky nav, welcome board, event cards, calendar rows, discussion rows, member rows, learning path cards, and prompt cards.

## 11. Interaction Model
Mobile chips show active focus. Discussion rows update the welcome board copy so the page feels responsive.

## 12. Accessibility
Semantic sections, real buttons and links, readable contrast, visible labels, and large tap targets are used throughout.

## 13. Participation Strategy
The page gives members three explicit choices: join a conversation, attend an event, or meet members.

## 14. Implementation Notes
The skeleton is standalone HTML with inline CSS and JavaScript, ready for catalog preview and later framework migration.
