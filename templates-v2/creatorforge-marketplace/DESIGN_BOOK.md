# CreatorForge Marketplace Design Book

## 1. Template Identity
CreatorForge Marketplace is a bold creator economy discovery homepage for brands booking creators by commercial fit.

## 2. Brief Match
- Type: creator economy marketplace.
- Layout: discovery homepage with category pills, featured creator cards, service bundles, social proof, and booking CTA.
- UX: creators are scannable by style, price, availability, and audience fit.
- Responsive: mobile uses vertical creator cards and sticky filter chips.

## 3. Audience
Growth leads, brand managers, agencies, and founders who need to shortlist creators quickly.

## 4. Information Architecture
The page starts with a marketplace promise, then a featured creator match, sticky filters, creator cards, bundles, proof, and booking CTA.

## 5. Layout System
Desktop uses a bold split hero and four-column creator grid. Mobile collapses into vertical discovery with sticky chips.

## 6. Responsive System
Below 760px the hero, creator cards, bundles, proof, and booking CTA become single-column and touch-friendly.

## 7. Typography
Syne is used for expressive headings and creator names. DM Sans is used for UI, descriptions, buttons, chips, and metadata.

## 8. Color System
- Black: #080808
- Hot pink: #FF3D9A
- Neon yellow: #F9F871
- Electric blue: #3B82F6
- White: #FFFFFF
- Lavender: #C4B5FD

## 9. Visual Language
The aesthetic is loud, high-contrast, and creator-forward, with generated portrait panels instead of stock photography.

## 10. Components
Core components include sticky navigation, hero match board, category chips, creator cards, bundle cards, proof strip, and booking panel.

## 11. Interaction Model
Filter chips show active selection. Creator cards update the featured match copy to make browsing feel responsive.

## 12. Accessibility
Landmarks, visible text labels, strong contrast, touch-sized controls, and semantic links/buttons are used throughout.

## 13. Commercial Hierarchy
Each creator card exposes fit score, category, price, audience, and availability so comparison does not require opening detail pages.

## 14. Implementation Notes
The skeleton is standalone HTML with inline CSS and JavaScript, ready for catalog preview and framework migration.
