# Prism Legal Tech Design Book

## Concept
Prism Legal Tech is a premium legal practice management skeleton for firms that need fast visibility across cases, documents, deadlines, billing, and client relationships.

## Typography
- Headings use Manrope for confident legal operations hierarchy.
- Interface text uses Inter for dense tables, navigation, and status metadata.
- Fonts load through the Google Fonts CDN.

## Color System
- Cloud `#F6F8FC` for the page surface.
- Ink `#19212B` for primary text, structure, and dark command panels.
- Indigo `#4F46E5` for primary actions and selected states.
- Cyan `#06B6D4` for live indicators and active event markers.
- Lime `#A3E635` for healthy, approved, and billable states.
- Coral `#FB7185` for urgent deadlines, alerts, and nonstandard billing states.

## Layout
The interface opens as an operational dashboard rather than a marketing page. A sticky top navigation gives immediate access to Dashboard, Cases, Documents, Calendar, Billing, and Clients. The primary content moves from firm-wide KPIs into practical work surfaces with case cards, document generation, calendar planning, billing sync, and client summaries.

## Components
- Sticky navigation with search, notification badge, and mobile hamburger drawer.
- KPI strip for active cases, deadlines, unbilled hours, and invoices.
- Case cards with matter number, client, type badge, status, owner, deadline, and progress.
- Document assembly table with template metadata and generate actions.
- Monthly calendar with event dots and a deadline sidebar.
- Billing table with hours, rate, billable status, and invoice actions.
- Client cards with active case count and billed totals.

## Accessibility
The skeleton uses semantic landmarks, descriptive labels, keyboard focus styles, a keyboard-navigable mobile drawer, Escape-to-close behavior, focus trapping inside the drawer, and reduced-motion handling.

## Responsive Behavior
Desktop uses a broad dashboard layout with multi-column grids. Tablet screens collapse navigation into the drawer and reduce card grids. Mobile screens use single-column content, compact calendar cells, and a slide-in drawer for navigation.
