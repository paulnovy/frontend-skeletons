Usage Guidelines

Rules are written for everyday product decisions: contrast, hierarchy, color intent, and interaction clarity.

Do

Use indigo for primary commitment.

Reserve the strongest action color for the next meaningful step.

Create release uses indigo.
Don't

Use every accent at once.

Multiple saturated signals make priority harder to scan.

Avoid mixing alert, status, and action in one row.
Do

Keep labels visible.

Fields should remain understandable after a user enters data.

Workspace name sits above the input.
Don't

Hide state in color alone.

Pair color with text, icon, or structure for accessibility.

Critical alert includes a label and message.
Do

Use 4px rhythm.

Spacing tokens create predictable density across docs and apps.

Controls use 8px or 12px internal gaps.
Don't

Stack framed containers.

Frames inside frames dilute hierarchy and add visual noise.

Use one surface per repeated item.
