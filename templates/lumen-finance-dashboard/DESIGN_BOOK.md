# Lumen Finance Dashboard Design Book

## Skeleton name and project type
Lumen Finance Dashboard is a premium Personal finance management skeleton in the dashboard category.

## Intended audience
Individuals and households who want a calm, accurate view of spending, savings, and investments.

## Main user problem solved
Makes cash flow, budgets, subscriptions, and portfolio changes easy to understand without spreadsheet maintenance.

## UX assumptions and user flow
Users arrive needing orientation first, then specific action. The flow moves from brand and purpose, to metrics, to working cards, to activity detail, to a trend or commercial proof section.

## Page structure description
The page includes mobile navigation, desktop navigation, a high-impact hero, executive metrics, an operational card grid, activity and decision panels, a chart or product proof area, and a footer with credit.

## Key components list
- Responsive hamburger navigation
- Brand lockup and anchor navigation
- Hero surface with primary and secondary calls to action
- Metric cards with contextual labels
- Status cards with progress meters and CSS-only sparklines
- Compact activity rows
- Decision pipeline panel
- CSS-only chart or specialty product section
- Footer with product links and credit

## Visual mood rationale
The mood is clean white and cool gray with confident emerald progress cues. It is designed to feel credible enough for a funded prototype, polished enough for an agency presentation, and practical enough for frontend implementation.

## Color usage description
Primary accent: `#10b981`. Secondary accent: `#047857`. Accent color is reserved for progress, state, action, and key values. Background and surface colors establish depth and domain tone.

## Typography choices
Inter is used for interface clarity and commercial polish. JetBrains Mono is available for numeric, technical, and system-like details.

## Interaction rules
Buttons, cards, and rows use subtle elevation, border changes, and visible focus states. Navigation preserves anchor behavior. Motion is short and disabled for users who prefer reduced motion.

## Responsiveness approach
The layout is mobile-first. It begins as a single column, expands to two-column hero and card grids, then adds desktop sidebar or landing navigation above 980px.

## Accessibility notes
The skeleton uses semantic landmarks, a skip link, descriptive labels, keyboard focus styling, large touch targets, and non-color text labels for state.

## Potential commercial use cases
- Investor or customer-facing prototype
- Agency design direction
- SaaS dashboard starter
- Product discovery workshop
- Internal tool redesign baseline
